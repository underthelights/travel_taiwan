### 🚄 타이중 ↔ 타이베이 이동 (조사 완료 — 정하는 중)
공항이 HSR역과 떨어져 있어 **2단계**:
1. **RMQ 공항 → HSR 타이중역(신우일/Wuri)**: 156번 버스 직통 40~50분(가장 쌈) / 택시·픽업 NT$850~1,200, 30~40분
2. **HSR 타이중역 → 타이베이**: 고속철 약 59분, 20분 간격, 편도 ≈NT$700(₩27,900). 신주는 중간역(타이중에서 ~20분). 외국인 할인은 클룩/KKday 사전구매가 쌈
- 귀국날 역산: 17:30 비행 → 15:00 공항 → 13:00 타이베이 출발

### 교통카드·패스 판단 (2026.06.27 조사 — 미확정)
- 한국 기후동행카드에 가까운 월정액은 대만 `TPASS/Megacity Pass` 계열이지만 30일권(NT$1,200)이라 2박3일 여행에는 불필요.
- 도쿄 메트로패스처럼 단기 무제한권은 `Taipei Metro 24/48/72hr`, `Taipei FunPASS Transportation` 등이 있으나, 현재 루트는 MRT 탑승 횟수가 많지 않아 대체로 손해.
- 고속철 무제한권(`THSR Pass`, `Taiwan PASS HSR`)은 남부 도시까지 길게 돌 때 적합. 이번 타이중→신주→타이베이→타이중 예상 HSR 합계는 정가 약 NT$1,370이라 패스 가격을 못 넘김.
- 현재 기준 추천 조합: **EasyCard 충전식 + THSR 외국인 구간권**, 귀국날 타이베이→타이중만 시간 지정.

## 3. 데이터 하네스 (`data.js`가 단일 소스)

### 3-1. 상태 모델 — 모든 장소/결정은 3단계 중 하나
| 상태 | 의미 | 표시 |
|------|------|------|
| `confirmed` | 확정 | ✅ 초록 — 사용자가 정한 것 |
| `candidate` | 후보(잠정) | 🟡 주황 — 제안만 된 상태 |
| `tbd` | 미정 | ⬜ 회색 — 자리만 잡음 |

- 장소 상태는 `confirmedPlaceIds` / `tbdPlaceIds`로 결정 (기본 candidate).
  현재 confirmed = `[1, 18]` (= 타이중 도착 / 출국, 항공권에 묶임).
- `window.placeStatus(id)` 헬퍼로 조회.

### 3-1b. 장소 분류축 `kind` — 메뉴 세분화 ([일차] × [분류])
각 장소는 `kind` 필드를 가짐. `placeKindMeta`의 6종 통제값 (자유서술 `cat`은 표시용, `kind`는 필터·그룹용):

| kind | 라벨 | 포함 |
|------|------|------|
| `flight` | ✈️ 비행기 | 공항 도착·출국 (항공) |
| `move` | 🚄 이동 | 고속철·MRT·버스·택시 환승 |
| `food` | 🍜 식 | 식사·야시장·디저트 |
| `stay` | 🏨 숙소 | 체크인·숙박 |
| `sight` | 🗼 관광 | 명소·사원·전망대·산책 |
| `shop` | 🛍️ 쇼핑 | 쇼핑·기념품 |

- **상단 메가메뉴**(`nav.js`): `[1·2·3일차]`에 hover(데스크톱)/캐럿 탭(모바일) → 분류별 드롭다운(개수 포함) → 클릭 시 `day.html?d=1&kind=food`로 이동. 모든 페이지 공통(DOMContentLoaded에서 TRIP 사용). 드롭다운은 body에 fixed로 부착(nav 클리핑 회피).
- `day.html`: `?kind=` 파라미터 있으면 타임라인·지도·상세를 해당 분류만 필터. 페이지 내 칩 바로도 전환 가능.
- 행정 절차(EasyCard·환전·보험·비자 등)는 일정 장소가 아니라 → **`info.html` 체크리스트**로 분리 (장소 kind에는 두지 않음).
- 장소 추가 시 **`kind` 필수 입력**.

### 3-2. 결정 큐 `decisions[]` — "하나씩 정해가는" 엔진
각 항목: `{id, cat, status, prio, title, deadline, note, options[]}`
- `status`: `todo`(정할 차례) → `doing`(정하는 중) → `done`(확정)
- `cat`: 사용자 우선순위 4축 + 준비물 → `food`(미식) / `route`(동선·숙소) / `sight`(관광) / `budget`(예산) / `prep`(준비물)
- `prio`: 1=먼저, 클수록 나중
- `window.planProgress()` → `{done, total, pct}`

### 3-3. 사용자 우선순위 (추천·정렬 기준)
**미식 · 동선/숙소 · 관광 · 예산 — 네 축 균형.** 한쪽으로 치우치지 말 것.

## 4. 결정 워크플로우 (★ 핵심 규칙)
사용자가 무언가를 **확정**하면 항상 이 순서로 반영:
1. 해당 `decisions` 항목 → `status: 'done'`, `note`에 확정 내용 기록
2. 관련 **장소를 confirmed로** (`confirmedPlaceIds`에 id 추가) / 필요시 장소 추가·수정·삭제
3. 장소가 바뀌면 **`node build_kml.js`** 실행해 `taiwan_trip.kml` 재생성
4. 이 `CLAUDE.md`의 "현재 확정 상태"·진행 메모 갱신

**금지:** 사용자가 확정하지 않은 것을 `confirmed`로 올리지 말 것. 제안은 `candidate`까지만.

## 5. 파일 구조
| 파일 | 역할 |
|------|------|
| `data.js` | **단일 소스** — 항공·예산·장소·팁·상태·결정 큐 + 헬퍼. 데이터 변경은 여기서만 |
| `index.html` | 홈 — 요약 + 전체 지도 + 결정 진행률 |
| `day.html?d=1/2/3` | 일자별 — 지도 + 타임라인 + 구간 이동 |
| `taiwan-notes.html` | 대만 핵심노트 — 배경지식·도시 역할·현지 규칙 |
| `stay.html` | 숙소 — 타이베이 권역·후보 호텔·예약 체크 기준 |
| `eat.html` | 먹킷리스트 — `TRIP.eats` 렌더링 |
| `train.html` | 교통 — EasyCard·패스·THSR 구간권·이동 방식 |
| `info.html` | 준비물 — 서류·수하물·현금/통신·지도 저장 체크리스트 |
| `plan.html` | (예정) 정하기 보드 — `decisions` 큐 렌더링 |
| `style.css` | 공용 스타일 |
| `build_kml.js` | `node build_kml.js` → 구글 마이맵 import용 KML 생성 |
| `taiwan_trip.kml` | 구글 My Maps import 결과물 |

## 6. 작업 규칙
- 데이터는 **`data.js`만** 수정. HTML에 일정·숫자 하드코딩 금지(렌더링만).

- 한국어 UI. 장소는 한글명 + `nameEn` 병기.
